# Writing Canon — master index + mandatory protocol

**This file is the entry point for ALL outreach writing. Load it FIRST, every time.**

Never start outreach writing from scratch or from memory. Before writing anything, load the canon below and write *against* it. After writing, check the draft *against* it.

> Writing is one step of a larger gauntlet. The canon governs the DRAFT. Drafting is only valid AFTER the pre-draft gates and only DONE after the post-draft gates.

## The process spine (run every gate, every time)
scope + ownership → **dedup** (not already sent, not already enrolled, not on do-not-contact) → **persona + live-LinkedIn read** (judge persona on the LIVE headline, not enrichment) → **one verified company web-anchor fact** → **load this canon** → **draft to the house voice** → **per-draft QC (MQS gate)** → ONE batch review → a single **APPROVE SEND** → **autonomous send-all** → append a **MASTER_SENT_LIST row after every send**.

- Nothing sends without an explicit APPROVE SEND.
- Enrich-first before research; TAM-only prospecting (only accounts on the rep's approved target list, `{{ASSIGNED_ACCOUNTS_OR_TAM_SLICE}}`).
- **Batch-intake routing:** survey the rep's existing Apollo sequences (`{{SEQUENCE_IDS}}`) FIRST, then dispose each contact preferring a sequence with a manual email step > a sequence with an automated-template step > a LinkedIn step > a one-off manual email. If routing is unclear, ask ONE upfront multiple-choice question before drafting.
- **Sender = the rep's `{{SENDER_EMAIL}}` (their `.ai` address)** for all outreach; the `.com` address is only for a rare, direct, fully-customized reply to a warm/inbound prospect.
- **LinkedIn CR/InMail research gate:** one evidence file per candidate, captured live in the main context; every activity-based hook must trace to a verbatim quote or be dropped; 2nd/3rd-degree only; per-row traceability. JS-only connection sends on public profiles are banned — use the Sales Navigator real-mouse flow with character-for-character readback.

## Mandatory protocol
**BEFORE:** load canon #1-#4 + the channel formula. Confirm the structure. Research is for RELEVANCE only — never recited at them.
**WHILE:** ask never assert; no resume recap / "I noticed"; no feature-led headline; one named Testsigma proof; outbound CTA = one meeting ask, inbound CTA = engagement question; 75-99 words; no em dashes (subject line included); sign-off `Best,` newline `{{FIRST_NAME}}`.
**AFTER:** run the MQS 12-point gate; confirm zero hard-rule violations; apply the "could-anyone-tell" test. Only ≥9/12 drafts go to the rep for the single APPROVE SEND, and every send gets a MASTER_SENT_LIST row.
